# 修正后的轨道结构说明

## 问题回顾
之前的实现中，轨道文件夹结构不正确，导致：
1. 次级轨道不是同级的
2. 不同item的轨道组没有正确分隔

## 修正后的轨道结构

### REAPER文件夹层级规则
- `I_FOLDERDEPTH = 1`：开始一个文件夹
- `I_FOLDERDEPTH = 0`：正常轨道（不在文件夹开始/结束中）
- `I_FOLDERDEPTH = -1`：在父文件夹中（结束当前文件夹）

### 修正后的实现逻辑
对于每个选中的item：

1. **一级轨道**（父轨道）：
   - 名称：item的文件名
   - `I_FOLDERDEPTH = 1`（开始文件夹）
   - 分配独特颜色

2. **前4个次级轨道**：
   - 名称：`文件名_子轨道01` 到 `文件名_子轨道04`
   - `I_FOLDERDEPTH = 0`（在文件夹中）
   - 分配浅色变体

3. **第5个次级轨道**：
   - 名称：`文件名_子轨道05`
   - `I_FOLDERDEPTH = -1`（结束当前文件夹）
   - 分配浅色变体

4. **下一个item的一级轨道**：
   - 重新开始新文件夹（`I_FOLDERDEPTH = 1`）

## 使用步骤
1. 在REAPER中选中一个或多个视频item
2. 运行 `Arrange_track_sample_GUI.lua` 脚本
3. 点击"根据选中视频建立轨道"按钮
4. 系统会自动创建正确的轨道结构

## 验证方法
1. 观察轨道列表，每个item的轨道组应该独立折叠
2. 点击一级轨道左侧的文件夹图标，应该只展开该item的5个次级轨道
3. 不同item的轨道组之间应该有明显的分隔

## 如果还有问题
1. 重启REAPER清除缓存
2. 确保运行的是最新修改的脚本
3. 可以运行 `测试轨道结构.lua` 查看详细说明
